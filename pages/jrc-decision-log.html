JRC decision log

Document metadata - Document type: Decision and escalation register - Status: Open decisions log for transition from draft v1 to client-reviewed v2 - Purpose: Capture the decisions that need JRC confirmation or review input so the package can progress without hidden ambiguity - Primary source set: - Requirements traceability matrix - draft v1 - AI intervention-point register - draft v1 - Human oversight and approval gates - draft v1 - Synthetic use case proposal - draft v1 - Demonstrator evaluation plan - draft v1 - Integrated deliverable outline - draft v1 - V2 refinement plan

How to use this log

  • One row per material decision.
  • Use Status to distinguish open items from confirmed outcomes.
  • Use Impact to show what gets blocked if the decision is not taken.
  • If JRC input is helpful but not strictly required, mark the item as advisory rather than blocking in the status note.

Status values

  • Open: no decision yet
  • Pending review: submitted to JRC for response
  • Provisional internal assumption: temporary working position used to avoid idle drafting
  • Confirmed: agreed direction for v2
  • Closed: decision implemented in affected artifacts
