Purpose of this draft

This version is not a blank template. It is a first executable control matrix for the project as actually proposed: a methodology-first, governance-first package with a synthetic demonstrator and bounded AI support for preparatory SMCE work.

This page translates the proposal commitments into:

  • interpreted requirements
  • concrete delivery responses
  • named working artifacts
  • planned evidence for acceptance
  • unresolved items that should become JRC decision requests rather than hidden ambiguity

Working package assumptions used in this draft

These assumptions are introduced so the project package can operate as a real first draft rather than a generic shell.

  1. The proof-of-concept remains file-based and replayable, not integrated into live EC systems.
  2. The demonstrator will use one synthetic Better Regulation-style case with a manageable option set, evidence corpus, and review trail.
  3. Working synthetic case assumption for planning only:

    4. - Topic: improving access to targeted household energy support through different eligibility and delivery models. - Baseline: application-based access with fragmented information, burdensome proof requirements, and substantial non-take-up among eligible households. - Option A: improved information and assisted application support within the current model. - Option B: pre-filled or data-assisted eligibility verification with simplified application. - Option C: proactive or semi-automatic enrolment using verified administrative data with stronger safeguards and review controls.

  4. This synthetic case is a planning anchor only. JRC may replace it, but the control logic below should survive topic changes.
  5. Final scoring, weighting, trade-off choice, and policy framing remain human-owned even in the demonstrator.

Proposed acceptance evidence set

  • Integrated package assembled under ART-01
  • Controlled artifact set ART-02 through ART-16
  • At least 6 completed evidence sheets from the synthetic corpus
  • At least 1 draft and 1 revised criteria structure note
  • At least 6 rationale notes across baseline and options
  • One populated uncertainty register with at least 5 material entries
  • One full demonstrator run with prompt/action log and approval log
  • One evaluator memo using the demonstrator evaluation criteria

Immediate decisions to request from JRC

  1. Confirm or replace the provisional synthetic case topic.
  2. Confirm preferred review cadence and which checkpoints require direct JRC participation.
  3. Confirm acceptable proof-of-concept model/tool environment.
  4. Confirm expected final delivery packaging and editable-source expectations.
  5. Confirm log storage and whether logs remain annexed or separate.

Main unresolved items in this draft

  • Synthetic case topic is still a working assumption.
  • Named people are not yet assigned to reviewer roles.
  • Log storage, redaction, and retention are still undefined.
  • Final export formats and acceptance packaging are not yet confirmed.
  • Better Regulation crosswalk is implied but not yet formalized as a standalone annex.
