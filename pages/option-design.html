Option design

Option design explains how policy options should be constructed for EC impact assessment and for comparative work in SMCE. Good option design is not a cosmetic step before the “real” analysis. It determines whether the later comparison is meaningful, whether the Assessment structure and multi-criteria impact matrix can be populated coherently, and whether the final report can withstand scrutiny.

This page should be read with Problem definition, Baseline and counterfactuals, Criteria design, Impact pathways, Assessment structure and multi-criteria impact matrix, Stakeholder input guidance, and Sensitivity and robustness analysis.

Why option design matters

In impact assessment, options are the alternatives decision makers are actually choosing between. If those alternatives are vague, internally inconsistent, politically impossible, or quietly incomparable, then even a sophisticated analytical method will produce weak results.

Bad option design causes recurring downstream problems:

  • impacts cannot be attributed consistently because the intervention logic is unclear
  • criteria become hard to score because analysts do not know what the option really contains
  • comparison becomes biased because one option is more fully specified than another
  • stakeholder feedback becomes noisy because different people imagine different versions of the same option
  • sensitivity analysis becomes uninformative because the option set was unstable from the start
  • the Regulatory Scrutiny Board or other reviewers may conclude that the assessment is not comparing genuine alternatives

In short: weak options poison the whole appraisal.

What a policy option is

A policy option is a coherent package of action, or of no new action, that could realistically be pursued and assessed against the baseline.

In Commission practice, the option set usually includes:

  • the baseline or no-new-action scenario
  • one or more substantive policy alternatives
  • sometimes variants with different levels of ambition, stringency, scope, or instrument mix

In SMCE terms, options are the rows of the comparative exercise. They must therefore be defined clearly enough that each option can be assessed against the same criteria and by reference to the same baseline.

Core design principle: options must answer the problem

Option design should come after a disciplined Problem definition and before detailed scoring. The logic is:

  1. define the problem and its drivers
  2. establish the baseline
  3. set objectives
  4. identify feasible intervention logics
  5. translate those into a small set of comparable options
  6. test whether the options are distinct, feasible, proportional, and assessable

This sequence matters. If options are generated before the problem is clear, they often become recycled preferences rather than analytical alternatives.

How policy options are constructed in EC impact assessment

Under Better Regulation Guidelines and Toolbox, options should be designed as realistic alternatives for meeting the objectives while addressing the identified problem. In practice, this usually means varying one or more of the following:

  • policy instrument type, such as regulation, incentives, information, coordination, standards, funding, or a mixed approach
  • level of intervention, such as minimum harmonisation, stronger harmonisation, targeted action, or broad action
  • target group or scope
  • timing or phasing
  • governance and enforcement model
  • degree of flexibility left to Member States or regulated actors

A useful way to build options is to start from intervention logic rather than slogans. For each candidate option, specify:

  • what the option actually does
  • who is affected
  • what obligations, incentives, or capabilities change
  • what implementation mechanism is assumed
  • how the change is expected to reduce the problem relative to the baseline
  • what main costs, benefits, risks, and distributional effects are expected

That makes it easier to connect the option to Impact pathways and later to Criteria design.

How option design works in SMCE

In SMCE, option design must support structured comparison across multiple criteria while keeping social conflict, value pluralism, and distribution visible. That implies a few additional requirements beyond ordinary policy brainstorming.

Options should be:

  • clearly enough defined that different analysts would describe the same option in similar terms
  • broad enough to represent genuinely different policy directions
  • narrow enough that each option still has a coherent internal logic
  • comparable under one shared assessment structure
  • suitable for later equity and stakeholder analysis

The classic SMCE danger is to pretend that any list of policy ideas can be dropped straight into a matrix. It cannot. The option set needs prior design discipline.

Main quality tests for good options

Distinct

Options should represent genuinely different courses of action, not superficial relabellings.

Distinctness can come from differences in:

  • instrument choice
  • ambition or stringency
  • scope or target population
  • implementation architecture
  • allocation of responsibility between EU, Member States, agencies, firms, or citizens

Useful tests:

  • If two options produced almost identical impact pathways, are they really different options?
  • Can reviewers explain in one sentence what makes Option A different from Option B?
  • Would stakeholders likely experience the options differently in practice?

If the answer is no, the options may be duplicates disguised as alternatives.

Feasible

Options must be implementable enough to deserve analytical attention.

Feasibility includes:

  • legal feasibility
  • administrative and institutional feasibility
  • technical feasibility
  • enforceability
  • political plausibility at least at a minimally credible level
  • availability of delivery capacity, data, funding, or compliance infrastructure

This does not mean only politically easy options may be assessed. It means options should not depend on implausible powers, non-existent institutions, or magical compliance.

Proportional

The option set and the amount of detail should match the significance of the initiative.

In Commission practice, proportionality means:

  • not generating a bloated menu of minor variants that cannot be analysed properly
  • not collapsing a major strategic choice into only one token alternative
  • giving more analytical attention to the dimensions on which the options really differ
  • avoiding excessive detail where impacts are small or uncertain

A proportional option set is usually selective rather than exhaustive.

Comparable

Options must be comparable against a common baseline and common criteria.

That usually requires:

  • one shared baseline across the option set
  • one consistent time horizon unless variation is documented explicitly
  • comparable geographic and stakeholder scope
  • similar level of specification across options
  • clear treatment of implementation assumptions

If Option A is defined as a high-level principle and Option B as a fully worked delivery plan, the comparison is already distorted.

A practical workflow for building options

1. Start from the problem, not from preferred instruments

Use Problem definition to identify:

  • the core problem
  • its main drivers
  • where in the causal chain intervention is possible
  • who is affected
  • what happens under the baseline

This helps ensure that options target the actual mechanism of the problem rather than its symptoms alone.

2. Map intervention families

Before finalising options, it often helps to identify a broader long list of intervention families, such as:

  • information and transparency measures
  • market-based incentives
  • binding regulatory obligations
  • governance and coordination measures
  • support, capacity-building, or funding tools
  • combinations of the above

This long-list stage is useful because it reduces the risk that the short list is just a preselected favourite.

3. Narrow to a small analytic set

From the longer list, build a short set of options suitable for appraisal. Reasons for dropping candidates should be explicit, for example:

  • outside legal competence
  • clearly inconsistent with objectives
  • not feasible within the timeframe
  • analytically redundant with another option
  • too marginal to justify full assessment

4. Define each option at the same level of detail

For each remaining option, specify at least:

  • description of the intervention
  • target actors and scope
  • implementation mechanism
  • expected behavioural or institutional change
  • timing and transition assumptions
  • key uncertainties

This creates like-for-like specification.

5. Check comparability against the baseline and criteria

Before populating the matrix, ask:

  • Can all options be assessed against the same baseline?
  • Can all options be assessed against the same draft criteria?
  • Are the main trade-offs visible across the option set?
  • Is the baseline included explicitly in the comparison?

6. Review with experts and stakeholders

Stakeholder input guidance is especially useful here. Early stakeholder and expert review can expose:

  • missing options
  • false distinctness between options
  • implementation barriers analysts overlooked
  • options that are acceptable on paper but impossible in practice
  • bundled measures that should be separated

7. Freeze the option definitions before detailed scoring

Once the comparative analysis begins, option definitions should be stable enough that later scores are not based on shifting versions of the same option. If the option set changes materially, document the change and revisit the matrix where necessary.

Common design patterns for option sets

Baseline plus increasing ambition levels

This is common where the main question is how strongly to intervene.

Example structure:

  • baseline
  • light-touch option
  • moderate option
  • stronger option

This works well when the main differences are intensity, coverage, or enforcement strength.

Baseline plus different instrument logics

This is useful where the core question is not “how much?” but “by what mechanism?”

Example structure:

  • baseline
  • information-based approach
  • incentive-based approach
  • regulatory approach
  • mixed approach

This often produces stronger learning because it surfaces different causal pathways.

Baseline plus targeted versus broad scope

Useful where intervention may be directed at high-risk actors, vulnerable groups, specific sectors, or the whole population.

Sequenced or phased options

Useful when timing, transition burden, or institutional capacity are central to performance.

Hidden design issue: options are often bundles

Most policy options are not single measures. They are bundles of measures, assumptions, and implementation choices. That is not automatically a problem, but it creates risks.

A bundle becomes problematic when:

  • it combines too many unrelated components
  • the internal components pull in different directions
  • analysts cannot tell which component drives the impacts
  • the bundle differs in granularity from the other options
  • a preferred measure is hidden inside a larger package so it avoids direct comparison

A good rule is: bundle only what belongs to one coherent intervention logic. If a package contains separable strategic choices, consider turning them into separate options or clearly defined sub-variants.

Common mistakes

Fake options

A fake option looks like an alternative but is not a genuine choice.

Typical fake options include:

  • a strawman that is obviously unworkable and exists only to make the preferred option look reasonable
  • a cosmetic variant that differs only in wording, not substance
  • an option that is theoretically imaginable but impossible under basic legal or institutional constraints
  • an option that quietly assumes the same core mechanism as the preferred option while pretending to be different

Fake options weaken credibility because they do not support a real comparative judgement.

Hidden bundles

A hidden bundle occurs when one “option” quietly includes multiple contested choices that should have been made visible.

Examples:

  • combining stronger enforcement, new reporting, and funding support into one option without clarifying whether the package is being judged as a whole or as an inseparable intervention
  • embedding stakeholder-compensation measures in only one option, making distributional outcomes hard to interpret
  • including a major implementation-capacity upgrade in one option while treating delivery assumptions as external for the others

Hidden bundles distort both impact attribution and weighting.

Unequal specification

If one option is described in concrete operational terms and the others remain high-level, the detailed option will often appear either unrealistically burdensome or artificially well evidenced. Both are distortions.

Option drift during assessment

Teams sometimes change what an option means after seeing early results or stakeholder reactions. If this happens silently, the matrix stops referring to a stable object of comparison.

Smuggling objectives into options

An option should state what is done, not simply restate the objective.

Weak option label:

  • improve compliance and fairness

Stronger option label:

  • introduce a risk-based inspection regime with mandatory reporting for high-risk operators

Confusing instruments with outcomes

“Higher trust” or “better innovation” are not options. They are hoped-for effects. The option is the intervention expected to influence those effects.

Ignoring implementation assumptions

Many options look attractive only because delivery is underspecified. Implementation should not be left as a blank space.

Overloading the option set

Too many options create analytical clutter, especially when differences are minor. A short set of well-defined, policy-relevant options is usually better than a long menu of thin variants.

Designing options that cannot be assessed on common criteria

If each option requires its own custom metric, the set may not support a coherent SMCE comparison. That may signal that the options are too heterogeneous or too poorly framed.

Option design and the impact matrix

Assessment structure and multi-criteria impact matrix makes clear that the comparison only works if the rows are well specified. Option design therefore affects:

  • whether each criterion can be scored consistently
  • whether uncertainty can be described coherently
  • whether stakeholder-specific evaluation is meaningful
  • whether later weighting and robustness analysis are interpretable

A useful pre-matrix check is to create a short option specification sheet for every option covering:

  • intervention description
  • scope
  • implementation logic
  • main expected impacts
  • key affected groups
  • principal uncertainties

If these sheets look radically uneven, the option set is not ready.

Option design and stakeholder analysis

Stakeholders can help both too early and too late if the process is not managed well.

Used well, early stakeholder input can improve option design by revealing:

  • missing real-world constraints
  • overlooked implementation burdens
  • unintended effects on specific groups
  • options that appear distinct to analysts but not to affected actors

Used badly, stakeholder input can distort option design if:

  • each stakeholder demand becomes a separate option without analytical discipline
  • organised interests are allowed to define the whole option set
  • consultation occurs only after a preferred option is effectively locked in

The aim is not to let consultation write the options, but to use consultation to stress-test them.

Option design and proportionality in evidence gathering

Not all options need the same depth of evidence at all stages. Early screening can be lighter, but full appraisal requires enough specification that impacts can be compared fairly.

A practical approach is:

  • broad long-list screening with light evidence
  • short-list development with clearer specification
  • detailed appraisal only for the final comparable set

This is often the only manageable way to respect analytical proportionality.

How to tell when the option set is good enough

A workable option set usually has the following features:

  • the baseline is explicit
  • each option has a distinct intervention logic
  • all options are feasible enough to merit analysis
  • the main strategic choices are visible, not hidden
  • the options can be compared on a common set of criteria
  • implementation assumptions are stated
  • major affected groups can be identified for each option
  • the set is small enough to analyse properly but broad enough to show meaningful choice

Weak option design damages the whole assessment

If the options are poorly designed, later steps become fragile:

  • Criteria design becomes arbitrary because the options do not expose stable differences
  • Impact pathways become speculative because the intervention is not clear
  • Assessment structure and multi-criteria impact matrix becomes inconsistent because not all options can be scored the same way
  • Equity analysis becomes misleading because distributional effects are tied to vaguely defined packages
  • Sensitivity and robustness analysis tests a weak structure rather than a sound one

This is why option design is not just preparatory drafting. It is one of the core quality gates in policy appraisal.

Practical takeaway

For EC impact assessment and SMCE, a good option set is not the longest list or the most politically convenient list. It is a bounded set of realistic, clearly distinct, proportionate, and comparable alternatives linked to the Problem definition and assessed against a shared baseline.

The test is simple: can a reviewer see that these are real choices, understand what makes them different, and compare them on common criteria without hidden assumptions doing the work? If yes, the option design is probably strong enough for the rest of the assessment.
