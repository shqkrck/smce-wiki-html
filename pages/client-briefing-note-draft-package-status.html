Client briefing note - draft package status

Purpose

This note summarises the current first-draft package, the operating logic it follows, the immediate points that now need confirmation, and the recommended agenda for the next JRC review discussion.

Current draft package status

The project has moved from framework design into a populated first-draft package. The following core draft components are now in place:

  • Requirements traceability matrix - draft v1
  • AI intervention-point register - draft v1
  • Human oversight and approval gates - draft v1
  • Synthetic use case proposal - draft v1
  • Demonstrator evaluation plan - draft v1
  • Integrated deliverable outline - draft v1

Taken together, these drafts now provide a workable control spine for the assignment: requirements mapping, bounded AI-use design, human review gates, a candidate synthetic demonstrator case, evaluation criteria, and the structure for the final integrated submission.

Operating logic of the package

The package is built on a governance-first, methodology-first, review-first logic consistent with the contract reread.

In practical terms, this means:

  • the assignment is being treated as one integrated deliverable combining method and demonstrator;
  • AI is positioned only in bounded preparatory tasks such as structuring evidence, drafting notes, flagging gaps, and preparing review materials;
  • final judgement, scoring, trade-offs, approvals, and policy interpretation remain with human reviewers;
  • every substantive output is intended to remain draft until reviewed and logged;
  • traceability, uncertainty visibility, and replayable records are being treated as core design requirements rather than annex-only compliance features.

What remains to be confirmed now

The package is sufficiently developed for targeted client decisions. The main items still requiring confirmation are:

  1. Synthetic use case choice
  • The package now contains a developed synthetic use case proposal, but the broader draft set is not yet fully aligned on the final topic anchor.
  1. Reviewer model and checkpoint participation
  • The functional roles are defined, but named reviewers and the exact JRC participation points are still to be confirmed.
  1. Permitted model/tool environment
  • The package assumes a lightweight, file-based proof of concept, but the acceptable model and tool configuration still needs agreement.
  1. Logging, storage, and access approach
  • The draft package defines what should be logged, but not yet the final storage, retention, and any redaction approach.
  1. Final packaging preferences
  • Confirmation is still needed on the expected final submission format, annex structure, and editable-source expectations.

Top risks if unresolved

If these points are left open, the main risks are:

  • scope drift or rework if the synthetic use case is changed late;
  • governance ambiguity if reviewer roles and checkpoint ownership are not fixed early;
  • avoidable redesign if tool constraints are confirmed only after demonstrator artifacts are populated;
  • weakened auditability if logging and storage rules are clarified too late;
  • inefficient final assembly if packaging expectations are not agreed before the integrated deliverable is compiled.

There is also a near-term coherence risk: if the package proceeds without locking the synthetic case and operating assumptions, different draft components may continue to evolve against different planning anchors.

Recommended next meeting agenda

  1. Confirm the synthetic use case to be used for the demonstrator.
  2. Confirm the human reviewer model and which checkpoints require direct JRC review.
  3. Confirm the acceptable proof-of-concept tool/model environment.
  4. Confirm the logging and storage approach, including whether logs sit in annexes or as separate support files.
  5. Confirm preferred final package structure and delivery format.
  6. Agree the next review round scope: which draft components should now be refined to review-ready status first.

Recommended immediate outcome

The recommended immediate outcome of the next meeting is a small set of explicit decisions that freeze the demonstrator basis and package-control assumptions. Once those are agreed, the current draft set is in a strong position to move from draft v1 into review-ready working versions without major structural redesign.
