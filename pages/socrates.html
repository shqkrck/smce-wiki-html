SOCRATES

SOCRATES is the Joint Research Centre’s software tool for applying SMCE to ex ante policy appraisal. The name stands for Social multi-CRiteria AssessmenT of European policieS.

In this wiki, SOCRATES is best understood as the operational tool layer that helps turn an SMCE design into a structured, reviewable option-comparison workflow for EC impact assessment work.

What it is

SOCRATES is a specialised decision-support application designed for impact assessment problems. It implements the SMCE framework rather than replacing it. Its purpose is to help analysts structure an impact matrix, process mixed evidence, represent stakeholder perspectives, and test whether conclusions are robust.

The official JRC descriptions stress that SOCRATES is meant to support policy learning and transparency, not to automate policy choice. It is therefore closer to an analytical workbench than to an automatic recommendation engine.

Role in EC impact assessment and the SMCE workflow

Inside the Commission setting, SOCRATES fits mainly in the middle and late analytical stages described in Policy making process - impact assessment:

  • after the problem, objectives, baseline, and option set have been defined
  • when criteria and dimensions need to be made explicit
  • when evidence has to be organised consistently across options
  • when stakeholder perspectives need to be encoded for equity analysis
  • when rankings or comparative conclusions need robustness testing before reporting

This makes SOCRATES relevant for:

  • building and analysing the multi-criteria impact matrix
  • supporting the equity analysis fed by stakeholder input
  • running local and global sensitivity checks
  • generating outputs that can be interpreted in the final EC impact assessment narrative

In practice, Better Regulation Guidelines and Toolbox still define what an impact assessment must cover. SOCRATES only helps perform one part of that work more systematically.

Key components

Official JRC material presents three core components.

1. Multi-criteria analysis

This is the main comparative module. It requires analysts to define:

  • dimensions
  • objectives
  • criteria
  • weights or importance relations
  • option-by-criterion performance information

The impact matrix can contain quantitative and qualitative information, including ordinal or linguistic assessments. The point is to keep comparison possible without forcing every criterion into a single monetary metric.

2. Equity analysis

This module brings in social actors and stakeholder perspectives. It uses a separate social or equity matrix rather than mixing stakeholder opinions directly into the technical evidence matrix.

In the local workflow, this connects directly to Stakeholder input guidance and Equity impact matrix guidance. The equity analysis helps show:

  • where stakeholder groups converge or conflict
  • how options look from different actor perspectives
  • what the likely distributional tensions are
  • what a social compromise picture looks like

3. Sensitivity analysis

SOCRATES includes sensitivity and robustness functions to test whether results hold under changed assumptions. Official descriptions highlight both:

  • local sensitivity analysis: one-at-a-time changes, such as adding or removing criteria or changing weights
  • global sensitivity analysis: simultaneous variation across combinations of weights

This is the main safeguard against treating one ranking as more definitive than it really is. See Sensitivity and robustness analysis.

Practical use in this wiki’s workflow

A realistic use pattern is:

  1. follow Better Regulation logic to define the problem, objectives, baseline, and options
  2. use SMCE principles to design the dimensions, criteria, and evidence structure
  3. collect and organise criterion-by-option evidence
  4. collect stakeholder input where needed and build the equity matrix
  5. load the structured information into SOCRATES
  6. review the comparative outputs and robustness checks
  7. translate the results into a policy-readable narrative for Regulatory Scrutiny Board scrutiny

That means SOCRATES is most useful when the upstream analytical design is already sound. It cannot rescue weak option design, vague criteria, or poor stakeholder grouping.

Practical strengths

  • Gives a repeatable way to implement SMCE in real appraisal work.
  • Handles mixed quantitative and qualitative evidence.
  • Keeps equity analysis visible rather than burying it inside a single score.
  • Makes assumptions about criteria, weighting, and comparison more explicit.
  • Encourages robustness testing before conclusions are reported.
  • Produces outputs that can support a more traceable impact-assessment annex or narrative.

Cautions and limits

  • SOCRATES is not a substitute for Better Regulation requirements on problem definition, consultation, proportionality, or reporting.
  • Results are only as credible as the criteria design, evidence quality, and stakeholder representation fed into the tool.
  • Rankings should not be treated as mechanically decisive; they remain decision support.
  • Equity results depend heavily on how stakeholder groups are defined and how preferences are elicited.
  • Sensitivity testing is essential; without it, a technically neat ranking may still be fragile.
  • Any use in Commission work still has to be explainable in plain policy terms and defensible under Regulatory Scrutiny Board review.

Working interpretation for SMCE-related work

A useful division of labour is:

  • Better Regulation Guidelines and Toolbox = procedural rule set
  • SMCE = methodological framework
  • SOCRATES = operational software implementation
  • EC impact assessment = formal analytical product

Under that interpretation, SOCRATES is valuable when it increases traceability, comparability, and robustness without obscuring the policy judgement that remains with human analysts and decision-makers.
