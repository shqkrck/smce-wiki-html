Stakeholder input guidance

Document metadata

  • Source title: How to collect input from stakeholders for the Social Multi-Criteria Evaluation
  • Document type: Operational guidance
  • Version: 1
  • Date: 04/09/2024
  • Authors: N. Ostlaender, E. Basyte Ferrari, G. Munda
  • Project: Social Multi-Criteria Evaluation of Policy Options
  • Reference context: aligned with EC Better Regulation guidance on consultation

What this document is for

  • Explains when, why, and how stakeholder input should be collected for SMCE.
  • Connects ordinary EC consultation practice with the specific information needs of SMCE.
  • Distinguishes two main uses of stakeholder input:

    • - shaping the assessment structure for the multi-criteria analysis - supplying evaluations of policy options for the equity analysis

Key concepts

  • Stakeholders / social actors: groups whose values, concerns, interests, or implementation role matter for the policy.
  • Consultation strategy: planned approach for identifying stakeholders, deciding how they will be involved, and collecting usable evidence.
  • Assessment structure: dimensions, objectives, and criteria used to compare policy options in the multi-criteria analysis.
  • Equity analysis: structured comparison of how stakeholder groups evaluate each option.
  • Valid group representation: consultation results must credibly represent the view of each stakeholder group.

Inputs

  • Problem analysis and initial policy framing
  • Draft or initial policy options
  • Stakeholder map and consultation strategy
  • Existing consultation channels: Call for Evidence, public consultation, expert groups, exploratory outreach, workshops, focus groups, surveys
  • Better Regulation consultation principles and toolbox guidance

Outputs

  • Stakeholder map and grouped stakeholder categories
  • Refined assessment framework / criteria set for the multi-criteria analysis
  • Structured stakeholder evaluations of policy options for the equity impact matrix
  • Consultation record documenting who said what

Step-by-step operational guidance

  1. Map stakeholders early
  • Identify groups affected by, implementing, interested in, or knowledgeable about the policy.
  • Use the six identification tests:

    • 1. Who is directly impacted? 2. Who is indirectly impacted? 3. Who is potentially impacted? 4. Whose help is needed to make it work? 5. Who thinks they know about the subject? 6. Who will show an interest in the subject?

  • Also decide how groups will be involved and who may represent those who cannot participate.
  1. Group stakeholders analytically, not just administratively
  • Grouping should make perspectives distinguishable.
  • If there is strong variation within a group, investigate why and consider re-grouping.
  • Keep grouping consistent across consultation activities.
  • Typical examples: public authorities, academia, manufacturers.
  1. Use early consultation to shape the assessment structure
  • Start from an initial framework proposed by the lead DG based on the problem analysis.
  • Use early stakeholder input to identify meaningful dimensions, objectives, and criteria.
  • Relevant sources can include:

    • - Call for Evidence - public consultation - existing EC expert groups - exploratory consultation activities

  • Stakeholder concerns and preferences help test whether the proposed criteria capture what matters.
  1. Run targeted review of the draft framework
  • Once the initial structure exists, ask key stakeholders to review it.
  • Use interactive targeted formats where useful, especially workshops or focus groups.
  • Aim to collect feedback on missing, unclear, or poorly represented criteria.
  • Refine the framework using consultation results and expert advice on data feasibility.
  1. Collect stakeholder input for the equity analysis later in the process
  • This step comes after policy options are sufficiently defined.
  • Ask all stakeholder groups to evaluate all options, including the baseline scenario.
  • Suitable formats are structured and comparable, such as:

    • - closed-question surveys - targeted consultation questionnaires - workshop voting or similar structured elicitation

  • Use one common representation across all groups, e.g. ranking, +++/---, or very good/very bad.
  1. Ensure comparability and validity of responses
  • All stakeholder groups must receive the same information.
  • All groups must respond on all options.
  • Responses should represent the group’s opinion validly, not isolated individual views.
  • Document clearly which stakeholder group expressed which view.
  1. Re-run the exercise if options change
  • If a new option is added or existing options are modified, the equity-input collection should be repeated.
  • The exercise depends on stakeholders reacting to the actual option set under analysis.
  1. Feed outputs into SMCE products
  • Early consultation results support the assessment structure and criteria selection.
  • Later structured option evaluations feed the equity impact matrix.
  • Use the dedicated operational guides/templates for matrix construction.

What stakeholder input is used for in SMCE

  • For the multi-criteria analysis:

    • - reveal concerns and priorities - help select relevant criteria - review whether the assessment framework reflects different views

  • For the equity analysis:

    • - show how each stakeholder group evaluates each option - identify conflict, support, and possible implementation brick walls - make distributional and preference differences transparent in the decision model

Cautions and quality conditions

  • Distinguishable stakeholder perspectives are a prerequisite for the equity analysis.
  • The general public as one undifferentiated category is usually not useful for the SMCE equity matrix.
  • Group definitions should remain consistent across consultation stages.
  • “Who says what” must be traceable and documented.
  • Consultation timing matters: criteria-related input is useful early; option-evaluation input is useful only once options are well defined.
  • If some groups cannot participate directly, representation arrangements should be considered explicitly.
  • Use Better Regulation consultation principles for openness, participation, transparency, and credibility.
